# 第二章  人员安全和风险管理的概念

## 2.1 人员安全策略和程序
* 招聘新员工几个不同步骤：1.创建职责描述或岗位描述、2.设置工作级别、3.筛选 应聘者、4.招聘和培训最合适该职位的人。
* 理解雇用新员工对安全的影响：为实施恰当的安全计划，必须为职责描述、职位分类、工作任务、工作职责、防止串通、候选人筛选、背景调查、安全许可、雇佣协议和保密协议等设立标准。通过采用这些机制，可确保新员工了解所需的安全标准，从而保护组织的资产。
* 职责描述：定义了需要分配给员工执行工作任务的角色。定义了改制为的安全网络环境中需要访问的类型和范围。
* 职责分离: 把关键的、重要的和敏感工作任务分配给不同的管理员或高级操作员，确保没有能够危害系统安全的个人，防止串通。
* 最小特权：要求在安全的环境中，用户应获得完成工作任务或工作职责所需的最小访问特权。可以限制敏感信息的脆弱性。
* 岗位轮换：提供知识备份；降低欺诈、数据更改、盗窃、破坏和信息滥用的风险，还提供同级审计，防止串通。
* 强制休假：为了审计和核实员工的工作任务和特权，可使用一到两周的强制休假，强制休假能够轻易发现特权滥用、欺诈和疏忽。

### 2.1.1 候选人筛选及招聘
* 对岗位候选人的筛选基于职责描述定义的敏感性和分类。特定职位的敏感性和分类取决于担任该职位的人员有意或无意地违反安全规定造成的危害程度。
* 筛选方法:1. 背景调查；2.推荐信调查；3.学历验证；4.安全调查验证；5.社交网络账户审查。

### 2.1.2雇佣协议和策略
* 雇佣协议：该文件概述了组织的规则、限制、安全策略、可接受的行为和活动策略、职责描述的细节、违规行为和后果，员工担任改制为的时间长度。
* 保密协议(NDA)：用于防止已离职的员工泄露组织的机密信息。
* 非竞争协议(NCA)
* 雇佣策略规定解雇员工的程序，包括现场证人、禁用员工的网络访问权限和执行离职面谈等内容。还包括护送被解雇员工离开公司，并要求归还安全令牌、徽章和公司财产。

### 2.1.3 入职和离职程序
* 入职可包括培训，获得工作技能，调整做事方式，努力使员工有效融入现有的组织过程和程序。
* 解雇的程序应包括有证人在场、归还公司财产‘禁止网络访问、离职面谈和由人员护送离开公司。
* 离职面谈的主要目的是根据雇佣协议、保密协议和任何其他与安全相关的文件，对前雇员的责任和限制进行审查。

### 2.1.4 供应商、顾问和承包商的协议和控制
* 服务等级协议（SLA:Service-Level Agreement）
* 供应商、顾问和承包商的控制用来确定组织主要的外部实体、人员或组织的绩效水平、期望、薪酬和影响。这些条款在SLA文档或策略中规定。
* SLA:服务级别协议提到的问题：1.系统运行时间；2.最长持续停机时间；3.最大负载；4.平均负载；5.争端职责；6.故障切换时间。
* SLA通常还包括在无法维持协议规定的情况下约定的赔偿措施。

### 2.1.5 合规策略要求
合规是符合或遵守规则、策略、法规、标准或要求的行为。

### 2.1.6 隐私策略要求
* 定义：主动防止为经授权访问个人可能识别的信息；防止未经授权访问私有或机密信息；防止在未同意或之情的情况下被观察、监视或检查。
PII：个人身份信息
* 相关法案：1.健康保险流通与责任法案、家庭教育权力和隐私法案、金融服务现代化法案、欧盟指令95/46/EC、通用数据保护条例和支付卡行业数据安全标准。

## 2.2安全治理
* 安全的目的是在防止数据丢失或泄露的同时保持已授权的访问。发生损害、破坏和泄露数据或其他资源的可能性称为风险。
* 安全治理是与支持、定义和指导组织的安全工作相关的实践的集合。安全治理通常与公司治理和IT治理密切相关并有交集。
* 第三方治理: 由法律、法规、行业标准、合同义务或许可要求强制规定的监督制度。
* 第三方治理的重点是检验安全目标、需求、法规和合同义务的合规性。
* 文件审查是阅读交换材料并根据标准和期望进行验证的过程，文件审查的一部分是对业务流程和组织策略的逻辑和实际调查。
* 风险管理、风险评估和风险处置都是执行过程/策略审查所设计的方法和技术。

## 2.3理解并应用风险管理概念
* 实现风险管理目标的过程称为风险分析。风险分析包括：检查环境中的风险，评估每个威胁事件发生的可能性和实际发生后造成的损失，评估各种风险* 控制措施的成本，完成风险防护措施的成本/收益报告并向高级管理层汇报。对组织的所有资产进行估算、评估和估值。
* 为成功实施安全解决方案，必须改变用户行为，这些改变包括改变常规工作活动乃至遵守安全策略中规定的标准、指南和程序。行为的改变包括用户完成一定层次的学习。学习有三种公认的学习层次：安全意识、培训和教育。 
### 2.3.1风险术语
* 资产: 环境中需要保护的任何事物，包括业务流程或任务中用到的所有资源。
* 资产估值: 根据实际的成本和非货币性支出给资产指定的货币价值。
* 威胁:任何可能发生的、对组织或特定资产造成不良或非预期结果的潜在事件。
* 脆弱性:资产中的弱点，是防护措施或控制措施的弱点，或缺乏防护措施/控制措施。
* 暴露:指脆弱性会被威胁主体或威胁事件加以利用的可能性是存在的。暴露并不意味着导致损失的事件正在发生。仅表示如果存在脆弱性和能利用脆弱性的威胁，就可能发生威胁事件或出现潜在的暴漏。
* 风险:指威胁利用脆弱性对资产造成损害的概率 风险 =威胁 * 脆弱性，安全的整体目的通过消除脆弱性和防止威胁主体和威胁事件危机资产。
* 防护措施：指能偶消除脆弱性或能抵御一个或多个特定威胁的事物。是减轻或消除风险的唯一手段。
* 攻击: 指威胁主体对脆弱性的利用。
* 破坏：指安全机制被威胁主体绕过或阻止
* 渗透：指威胁主体通过规避安全控制获得对阻止基础设施的访问，并能能直接危及资产。
* 资产遭受威胁，威胁利用脆弱性，脆弱性导致暴露，暴露就是风险，而保护措施可以减轻风险，以保护资产的安全。

### 2.3.2 识别威胁和脆弱性
* 识别、检查、评估和阻止或减轻风险的过程称为风险管理。风险管理的目标使将风险降低到可接受的水平。

### 2.3.3 风险评估/分析
风险评估/分析主要是高层管理人员的工作。他们负责通过定义工作的范围来启动和支持风险分析和评估。
指定了威胁列表，就必须单独评估每个威胁及其相关风险。
**定量风险分析**
* 定量风险分析的过程从资产估值和威胁识别开始，接下来评估每个风险的可能性和发生频率。然后用这些信息计算用于评估防护措施的各种成本函数。
	* 1. 为每个资产分配资产价值(AV)。
	* 2. 暴露因子(EF)%：表示如果已发生的风险对组织的某个特定资产造成损坏，组织将因此遭受的损失百分比。
	* 3. 单一损失期望(SLE)(SLE=AV*EF):特定资产发生单一风险的相关成本。SLE以货币为单位。SLE = 资产价值(AV) * 暴露因子(EF)
	* 4. 年度发生率(ARO)(#/年):在一年内特定威胁或风险发生的预期概率。
	* 5. 年度损失期望(ALE)(ALE=SLE*ARO 或 ALE=AV*EF*ARO):是针对特定资产的所有可发生的特定威胁，在年度内可能造成的损失成本 ALE = SLE * ARO
	* 6. 计算使用防护措施时的年损失期望
	* 7. 计算防护措施的年度成本(ACS)($/年)：
	* 8. 防护措施的成本/收益：在成本/收益计算中结果值的最大控制措施，就是针对特定资产和威胁的特定风险所采用的特定防护措施的成本/效益公式：（ALE1-ALE2）- ACS 
* 评估防护措施时，每年节省或消耗的费用不应该是唯一考虑的因素，还应该考虑法律责任和应尽关心的原则。
* 要对防护措施进行成本/效益分析，必须计算出以下三个元素，得到针对特定资产的特定风险采用的特定防护措施的成本/效益公式：(ALE1 - ALE2) - ACS
	* ALE1：资产与威胁组合在控制措施实施前的ALE
	* ALE2：资产与威胁组合在控制措施实施后的ALE
	* ACS：防护措施的年度成本(ACS)($/年)

	
**定性风险分析**
采用的技术：1.头脑风暴；2.Delphi技术；3.故事板；4.焦点小组；5.调查；6.问卷；7检查清单。8.一对一的会议；9.面谈。
* 场景，对单个主要威胁的书面描述，包括威胁如何产生，以及可能对组织、IT基础结构和特定资产带来的哪些影响。
* Delphi技术:简单的匿名反馈和响应过程，不断重复，指导达成共识。
* 选择控制措施时风险评估或风险分析后的一项活动。

### 2.3.4 风险相应
* 风险分析的结果：
	* 1. 所有资产完整的、信息的估值。
	* 2. 包括所有威胁和风险、发生概率及造成损失成都的详细清单。
	* 3. 针对特定威胁的有效防护措施和控制措施列表。
	* 4. 每个防护措施的成本/效益分析。
* 对风险的控制措施：
	* 风险缓解：指通过实施防护措施控制措施以消除脆弱性或阻止威胁。
	* 风险转移：把风险带来的损失转嫁给另一个实体或组织。（购买保险或外包）
	* 风险接受：同意接受风险发生所造成的结果和损失。组织决定是否接受风险取决于组织的风险容忍度。风险容忍度也成为风险偏好。
	* 风险威慑：是对可能违反安全和策略的违规者实施震慑的过程。
	* 风险规避：选择替代的选项或活动的过程。
	* 风险拒绝：拒绝或忽视。否认风险的存在并希望永远不会发生。并不是合法的、正确的风险相应方式。
	* 残余风险：选择了控制措施，余下的风险就称为残余风险。选择接受而不去减轻。
	* 总风险计算公式：威胁 * 脆弱性 * 资产价值 = 总风险
	* 残余风险计算公式：总风险 - 控制间隙 = 残余风险

### 2.3.5 选择与实施控制措施
风险管理领域中选择控制措施或安全控制在很大成都上依赖于成本/效益分析结果。
* 控制措施包括预防、检测、修正、震慑、恢复、指示和补偿控制。控制也可分为管理性、逻辑性和物理性控制。
* 技术性控制措施：使用逻辑性或技术性控制措施。
* 技术控制措施示例：身份验证方法（如用户名、密码；智能卡和生物识别技术）、加密、限制端口、访问控制列表、协议、防火墙、路由器、入侵检测系统、阀值级别。
* 管理性控制措施：是依组织的安全策略和其他法规或要求而规定的策略和程序。包括策略，程序，招聘实践、背景调查、数据分类、数据标签、安全意识培训、休假记录、报告和审查、工作监督、人员控制和测试。
* 物理性控制：是可实际接触到的措施，包括组织、监测或检测对基础设施内系统或区域的直接接触的物理学控制措施。

### 2.3.6 适用的控制类型
* 1. 威慑控制：威慑潜在的攻击者。
* 2. 预防控制：避免意外事件的发生。
	* 威慑控制取决于个人决定不采取不必要的行动，预防控制阻碍了行动。
* 3. 检测控制：帮助识别意外活动和潜在入侵者。
* 4. 补偿控制：能提供可替代的控制措施。
* 5. 纠正控制：意外事件发生后修补组件或系统。
* 6. 恢复控制：使环境恢复到正常的操作状态。
* 7. 指令控制：指示、限制或控制主体的行为，以强制或鼓励遵守安全策略。

### 2.3.7 安全控制评估
* 安全控制评估(SCA)是根据基线或可靠性期望对安全基础设施的各个机制进行的正式评估。
* SCA的目标是确保安全机制的有效性，评估组织风险管理过程的质量和彻底性，并生成已部署的安全基础设施相对优缺点的报告。

### 2.3.8 监视和测量
安全控制提供的收益应该是可被检测和测量的。

### 2.3.9 资产估值与报告
* 风险分析的一个重要步骤是估算组织资产的价值。
* 评估资产的目标是为资产分配具体的货币价值。
* 资产估值是通过部署防护措施实现资产保护的成本/收益分析的基础，是选择或评估防护措施和控制措施的一种手段。
* 风险报告是风险分析的最后一项关键任务。风险报告包括编制风险报告，并将该报告呈现给利益相关方。
* 风险报告应能准确、及时、全面的反映整个组织的情况，能清晰和准确地支持决策者指定，并定期更新。

### 2.3.11 风险框架
风险框架是关于如何评估、解决和监控风险的指南和方法。
风险管理框架(Risk Management Framework, RMF)包括6个步骤：1.安全分类；2.选择安全控制、实施安全监控、评估安全控制、授权信息系统和监视安全控制。

## 2.4 建立和维护安全意识、教育和培训计划
* 实施安全培训的一个前提条件是建立安全意识。培训或教导员工执行他们工作任务并遵守安全策略中规定的标准、指南和程序。建立安全意识的目标是将安全放在首位并且让用户认可这一点。

## 2.5 管理安全功能
* 要管理安全功能，组织必须实现适当和充分的安全治理，通过执行风险评估以驱动安全策略的行为是安全功能管理最显著和最直接的示例。
* 安全必须可度量，安全管理功能包括信息安全策略的开发和实现。

4. 讨论进行“平衡的风险评估”的需求。可使用哪些技术？为什么需要这样做？
* 风险评估通常混合使用定量风险分析和定性风险分析。完全定量风险分析是不可能的，并不是所有的分析元素和内容都可以量化，应为有些是定性的，有些是主观的，有些是无形的。因此必须平衡定量分析的结果，在组织最后的风险评估中混合使用定量风险分析和定性风险分析的方法称为混合评估或混合分析。
